[PART 1: ARCHITECTURE & HOW IT WORKS / रचना आणि कार्यपद्धती]

1. THE THREE-PILLAR ARCHITECTURE:
   • The Bot (Automation): Uses 'Playwright' to control a Chromium browser. It acts 
     like a human user—clicking buttons, reading text from the screen, and 
     listening to the meeting.
   • The Backend (Control Room): A FastAPI server that stores all data in memory, 
     triggers AI tasks, and talks to the Frontend.
   • The Frontend (User Interface): A React dashboard that lets you control the 
     bot and see the results (transcripts, summaries, etc.).

2. HOW VOICE RECORDING WORKS (CORE LOGIC):
   • Step 1: Captures the system audio using 'PyAudio' in 30-second chunks (WAV files).
   • Step 2: Since raw recording can have noise, 'FFmpeg' is used to normalize 
     and clean the audio.
   • Step 3: These chunks are sent to 'Groq Whisper' API. Whisper converts the 
     sound into text (Transcription).
   • Step 4: The text is merged with previous chunks to create a full meeting transcript.

3. HOW SCREENSHOTS & VISION WORK:
   • The Bot continuously monitors the Google Meet DOM (HTML code) for the 
     "Screen Sharing" indicator.
   • When detected, it triggers a 'screenshot' command in Playwright.
   • These images are sent to 'Llama 4 Vision'. The AI "sees" the slide, extracts 
     the text, detects dates, and identifies key points on the screen.

1. DIRECTORY STRUCTURE:

     browser is controlled.
   • /backend/app: The "Logic". 
     - 'main.py' contains the API endpoints.
     - '/services' contains the AI integrations (Groq, OCR, Vision).
   • /frontend/src: The "Face". Look at 'App.tsx' to see how it fetches data 
     from the backend.

2. HOW TO START CODING THIS FROM SCRATCH:
   • Phase 1 (The Body): Start with Playwright. Just write a script to open 
     Google Meet and join a link.
   • Phase 2 (The Ears): Add PyAudio recording. Save audio files to a folder.
   • Phase 3 (The Brain): Connect to Groq API to convert those files into text.
   • Phase 4 (The Memory): Create the FastAPI backend to store and serve the text.
